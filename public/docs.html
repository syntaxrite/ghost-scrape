API Documentation

Ghost Scrape API

Turn public web pages into clean, LLM-ready Markdown for agents, crawlers, and developer workflows.

Auth
x-api-key
Use your API key in a request header.
Format
POST JSON
Send a URL in the JSON body.
Output
Markdown
Returns clean text, title, source, and word count.

Official request

POST /api/scrape
Content-Type: application/json
x-api-key: ghost_xxx

{
  "url": "https://example.com/article"
}

For the public UI demo, /api/demo also accepts the same URL body, but POST JSON is the official developer contract.

Quick curl test

curl -X POST "https://ghost-scrape.tech/api/scrape" \
  -H "Content-Type: application/json" \
  -H "x-api-key: ghost_xxx" \
  -d '{"url":"https://example.com"}'

If you are testing the demo flow without a key, call /api/demo with the same JSON body.

Response shape

{
  "success": true,
  "title": "Article title",
  "domain": "example.com",
  "source": "axios",
  "wasBlocked": false,
  "wordCount": 1234,
  "markdown": "# Clean content ..."
}

JavaScript example

const res = await fetch("https://ghost-scrape.tech/api/scrape", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": process.env.GHOST_API_KEY
  },
  body: JSON.stringify({
    url: "https://example.com"
  })
});

const data = await res.json();
console.log(data.markdown);

Python example

import os
import requests
from dotenv import load_dotenv

load_dotenv()

res = requests.post(
    "https://ghost-scrape.tech/api/scrape",
    headers={"x-api-key": os.getenv("GHOST_API_KEY")},
    json={"url": "https://example.com"}
)

print(res.json()["markdown"])

For agents

Ghost Scrape is built as a preprocessing layer for search-driven agents, RAG pipelines, and crawler workflows. Use it when search results return noisy HTML, navigation clutter, or page boilerplate, and you need clean Markdown for downstream models.
